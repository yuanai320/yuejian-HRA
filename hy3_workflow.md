# 我用 Hy3 构建「体检解读知识库」的工作流（= 可复用的 Skill / Agent 设计）

> 这份文档本身就是面试作品的一部分：它证明我能**把内容生产流程产品化**，
> 对应目标岗位 JD 中「把内容生产流程沉淀成可复用的 Skill 或 Agent」。

## 工作流总览（5 步闭环）
1. **定义 schema**：先确定每条指标需要哪些字段（分层阈值 / 含义 / 运动·营养·生活·就医建议 / 为什么这么分层）。
2. **AI 批量生成**：把 schema 喂给 Hy3（元宝），让它按临床指南生成初稿。
3. **人审把关**：我用专业知识（指导员证 + 营养师备考 + 慢病基础）校验医学准确性，删改错误。
4. **编码沉淀**：把审过的知识写进 `knowledge_base.py`，改数据不改代码（可复用）。
5. **迭代扩展**：新指标 / 新人群（如孕产妇、老年）用同一 prompt 模板增量生成。

## 为什么这是「AI Native」而不是「用 AI 写了一段文案」
- **可复用**：prompt 模板固化（见 `prompts_for_yuanbao.md`），换指标即可批量生产。
- **人机分工清晰**：AI 做信息整合与初稿，人做医学把关与用户体感——这正是 JD 要的"判断哪些适合 AI、哪些适合人"。
- **产品化输出**：最终不是一段对话，而是一个**能跑、能部署、能演示**的产品。

## 可迁移到岗位的同类 Skill（面试时主动讲）
- 「运动课程内容生成 Skill」：输入目标/基础/器械 → 输出带编排逻辑的课程（同构）。
- 「用户反馈分析 Skill」：喂入 App 评论 → 输出痛点聚类与内容优化建议。
- 「竞品分析 Skill」：喂入竞品功能清单 → 输出差异点与机会点。

> 一句话：我交付的不是"一个答案"，而是一套**能用 AI 持续产出的方法论**。
