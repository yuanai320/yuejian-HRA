# 在元宝 / WorkBuddy 直接跑的 Prompt 模板（Hy3）

> 用途：你亲自用 Hy3 扩充知识库、产出作品内容，留下"我用 AI 做"的真实证据。
> 把生成结果复制进 `knowledge_base.py` 对应字段即可。

## Prompt 1：批量生成某指标的解读（示例：甲状腺功能）
```
你是一位资深健康管理师。请为「促甲状腺激素 TSH」设计体检解读知识库条目，JSON 结构如下：
{
  "label":"促甲状腺激素TSH","unit":"mIU/L",
  "why":"用一句讲清为什么这么分层（用户体感角度）",
  "levels":[
    {"min":下限,"max":上限,"level":"分级名","note":"含义（用户看得懂）",
     "actions":{"运动":[...],"营养":[...],"生活":[...],"就医":[...]}}
  ]
}
要求：阈值参考国内指南通用值；干预建议具体可操作；分正常/临界/异常三档以上。只输出 JSON。
```

## Prompt 2：生成 HRA 风险分层逻辑
```
我有一份 HRA（健康风险评估）报告，含「器官衰老年龄」「慢病风险分数」「十大系统失衡」。
请设计一套「风险分级 → 干预优先级」的通用逻辑框架，输出为可编码的规则描述
（例如：风险分≥70 且涉及心血管 → 优先运动+营养干预，并建议就医）。要专业、可落地。
```

## Prompt 3：把一份真实体检异常项变成个性化方案
```
我是健康管理员。用户：男/22岁，体检异常：收缩压138、空腹血糖6.3、BMI26.5、尿酸460。
请基于「运动+营养+生活」三维，给一份 4 周可执行的改善方案，说明每周重点与风险点。
```

## Prompt 4：竞品分析（产品脑展示）
```
请对比三款「体检报告 AI 解读」产品（如：体检机构官方解读、健康类 App、微信小程序），
从「解读深度 / 个性化 / 干预落地 / 交互体验」四个维度分析差异点与机会缺口，
并指出一个尚未被满足的用户需求。输出结构化对比表。
```

## 使用建议
- 每个 Prompt 跑完，把结果**截图留存**——面试时这就是你"AI Native 工作流"的实证。
- 生成内容务必**人工校验医学准确性**再进代码库。
- 把常用 Prompt 存成「我的 Skill 库」，下次换指标直接复用。
