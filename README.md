# 悦检 · AI 体检报告解读与干预助手（v3.5）

> 一份能跑通、能演示、能部署的「健康服务 × AI」作品，用于投递 **AI × 运动健康 / 健康管理** 类实习岗位。

## 一句话定位
用户输入 HRA 体检报告 → 系统对照内置专业知识库 → 输出**风险分级 + 逐项解读 + 六维统一综合干预方案（饮食/运动/睡眠/用药/情绪管理/生活方式）+ 就医/用药/趋势预警/健康宣教**的结构化报告。

## 为什么做这个（动机）
- 我是**健康服务与管理**专业学生，持**社会体育指导员证**、正在**备考公共营养师**，能解读体检报告。
- 看到 vivo 运动健康研究所、「运动内容产品（AI×运动健康）」等实习岗，意识到 **"专业纵深 + AI 工具 + 产品化"** 是这类岗位的复合门槛。
- 借 **腾讯混元 Hy3 限免期（2026-07，至 7/22）**，用 AI 把"体检解读"这一最硬的专业能力，做成一个可复用的产品雏形。

## AI 在这里的角色（透明的 AI Native 工作流）
- 知识库中的**分层逻辑 / 指标含义 / 干预文案**，由我在 **元宝 / WorkBuddy** 中用 Hy3 辅助生成与校验；
- 我负责**医学准确性把关**与**用户体感打磨**——AI 规模化生产内容，人做质量闸门；
- 整个流程沉淀为 `hy3_workflow.md`（= 可复用的「解读 Skill / Prompt 工作流」），对应岗位 JD 中 **"把内容生产流程沉淀成可复用的 Skill 或 Agent"**。

## 技术栈
- Python 3 · Streamlit（交互界面）· 纯内置知识库（离线可跑，无需 API key）

## v2 功能（已用真实 HRA 电子数据校准）
1. **两段式流程**：先做生活方式问卷（10 题）→ 再进入报告解读页；
2. **五入口导入**（怎么方便怎么来）：
   - A. **粘贴报告文本**：PDF/网页复制，或 Excel 中只复制区域列；自动识别会员信息与区域检测值；
   - B. **上传 Excel 数据文件**：直接上传惠斯安普 HRA 导出的 `.xls / .xlsx / .csv`，pandas 原生读取，按记录选择解读（重度多行单元格也不错位）；
   - C. **上传 PDF 报告**：自动提取文本并解析区域检测值（pypdf，需带文字层；纯图片 PDF 请用 E）；
   - D. **上传 Word 报告**：自动提取文本与表格并解析区域检测值（python-docx）；
   - E. **📷 拍照/图片识别**：纸质报告拍照上传（jpg/png），用 Tesseract 免费 OCR 识别中英文后自动解析（无需 API key，Streamlit Cloud 通过 packages.txt 装中文包）；识别率低时提示改用带文字层 PDF / Word。
   - 文档类（PDF/Word）额外做了「区域名与值分两行」的容错解析，提升识别率。
3. **真实样本一键试用**：内置真实电子数据样本（T6，女/160cm/58kg），点「填入真实样本」即可看到端到端结果；
4. **逐系统局部解读**：呼吸/消化/肝胆/免疫/内分泌/泌尿生殖/耳鼻喉/心血管/神经/骨骼 **十大系统**、共 **126 个身体区域**（已与真实导出表头 100% 对齐）的风险分级与当前状态；
5. **综合研判（矛盾论框架）**：从所有系统中揪出**主要矛盾**、该系统内的**主要方面**与**次要方面**，并指出次要矛盾；
6. **🎯 行动优先级 Top 3**：跨维度直接告诉用户「最该先做好哪三件事」，避免面对一堆建议无从下手；
7. **六维统一综合干预建议**：饮食 / 运动 / 睡眠 / 用药 / 情绪管理 / 生活方式——按「主要矛盾系统优先」合成**统一方案**，不逐系统罗列，并给出优先级（①最优先 / ②优先 / ③协同）；饮食维度另附**推荐食物（含作用）、推荐茶饮、饮食禁忌、用餐习惯**明细；
8. **🏥 就医建议**：每个异常系统给出「建议挂什么科 + 推荐检查项目」；
9. **💊 用药建议**：强调**务必遵医嘱**，不可自行加量/减量/停药，并逐系统给出用药注意事项；
10. **⚠️ 身体状态趋势预警**：说明「若不干预会怎样、可能发展成什么疾病/后果」；
11. **📖 健康宣教科普**：每个系统一句话的医学科普；
12. 各系统风险评分**横向条形图**（系统名正排可读，不再倒转）+ 一键下载 Markdown 报告。

### v3 新增（对标世界一流体检/健康平台 + 7 导师评审迭代）
> 对标 **Function Health / InsideTracker / Mayo·Cleveland Clinic 高管健康** 的体验设计，并吸收「马斯克·乔布斯·纳瓦尔·Jack要加油·摩根·豪泽尔·罗伯特·清崎·博多·舍费尔」七位导师的评审建议：
13. **🏠 一屏看懂（首页概览）**：打开报告第一屏即见「综合健康指数 + 一句话解读」「⚡60 秒行动」「🎯首要目标」「🔴主要矛盾」——抓人且决策友好（对标生物年龄 / InnerAge，乔布斯极简）；
14. **综合健康指数（北极星指标）**：由「异常区域占比 + 最大功能偏离」折算 0–100 分与「健康/良好/亚健康/警戒」四级（≥85 健康 · 70–84 良好 · 55–69 亚健康 · <55 警戒），一眼判断整体健康水位；
15. **📡 系统风险雷达图**：matplotlib 雷达图把"哪里弱"一眼呈现（对标 InsideTracker Body Map）；
16. **🟢🟡 健康资产 / 负债视图**：正常系统=资产（保护），异常=负债（优先偿还），用资产思维讲优先级（罗伯特·清崎）；
17. **🎯 目标导向**：问卷新增「首要健康目标」，报告据此做目标连线（纳瓦尔 / InsideTracker goal-based）；
18. **📝 行动打卡（行为闭环）**：Top3 + 六维首条转为可勾选清单 + 成功日记提示，把"知道"变成"做到"（Jack要加油 / 博多·舍费尔）；
19. **📅 复测与跟踪（健康储蓄账户）**：依主要矛盾风险给复测周期，并可「保存本次为基线 JSON」下次复测对比（摩根·豪泽尔 / Function Health / InsideTracker 趋势追踪）；
20. **📎 就诊出示本报告**：就医建议强调携带报告便于医生快速了解风险分布（Function Health 医生协作）。

### v3.1 新增（库兹韦尔·塔勒布·芒格·卡尼曼·格鲁夫·老子 六位导师评审迭代）

## 运行方式
```bash
pip install -r requirements.txt   # 或逐包：pip install streamlit pandas xlrd openpyxl pypdf python-docx
streamlit run app.py      # Windows 未把 Python 加 PATH 时用：py -m streamlit run app.py
```
浏览器先填问卷 → 粘贴报告（或「填入示例 / 真实样本」）/ 上传 .xls/.pdf/.docx → 生成综合解读 → 下载报告。
